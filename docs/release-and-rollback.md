# 发布与回滚说明

## 灰度启用策略
1. 首先在低风险命名空间启用 webhook 接入与对象级熔断。
2. 执行主线A演练脚本，确认三项强断言通过。
3. 观察关键指标（失败率、回滚次数、熔断次数）24 小时。
4. 通过配置开关启用域级熔断。

## 风险
- 健康 Revision 误判导致回滚不准确。
- 域级熔断阈值过严导致自愈过度抑制。
- 告警重复风暴导致链路抖动（需依赖幂等窗口）。

## 回滚步骤
1. 将策略模式切换为 `L1 + 人工介入`。
2. 关闭 webhook 自动写入，仅保留只读评估与审计。
2. 关闭域级熔断开关，仅保留对象级熔断。
3. 如果仍异常，停用自动写操作，仅保留只读评估与告警。
4. 根据审计记录恢复到最近稳定发布版本。

## 应急步骤
- 持续失败达到阈值时，立即触发人工介入。
- 对生产命名空间临时启用维护窗口，阻断自动写操作。
- 当证据不足导致频繁 L3 时，暂停 L2 并执行人工版本选择。
