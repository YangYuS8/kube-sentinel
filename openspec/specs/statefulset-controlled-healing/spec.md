## 目的

定义 StatefulSet 受控自动动作的启用边界、四重授权门禁与失败回退冻结机制，确保在可控风险下提升恢复效率并维持人工兜底。

## 需求

### 需求:StatefulSet 受控自动动作开关

系统必须为 StatefulSet 提供默认关闭的受控自动动作开关；仅在开关启用时允许进入可写执行判定。

#### 场景: 开关未启用

- **当** StatefulSet 触发自愈链路且受控开关为关闭
- **那么** 系统必须保持只读阻断并输出明确原因

### 需求:StatefulSet 四重授权门禁

系统执行 StatefulSet 自动动作前，必须同时满足配置开关、命名空间白名单、对象级审批标记与运行态证据完整性。

#### 场景: 任一授权条件不满足

- **当** 四重授权门禁中的任一项失败
- **那么** 系统必须禁止写操作并回退到只读评估

### 需求:失败回退只读

系统在 StatefulSet 自动动作失败后，必须立即回退到只读模式并冻结后续自动重试，直到人工解锁。

#### 场景: 受控动作失败

- **当** StatefulSet 自动动作执行失败
- **那么** 系统必须记录失败证据、进入只读冻结状态并要求人工介入
