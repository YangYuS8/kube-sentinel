## 目的

定义生产环境中 StatefulSet Phase 2 的灰度发布、指标门禁与参数配置要求，确保可控启用、可观测回退与最小风险上线。
## 需求
### 需求:Phase 2 灰度发布控制

系统必须支持 StatefulSet Phase 2 的灰度发布控制，至少包含命名空间白名单、对象级审批与快速回滚开关。

#### 场景: 灰度范围受控

- **当** 运维仅对部分命名空间启用 Phase 2
- **那么** 系统必须确保非白名单命名空间继续保持只读策略

### 需求:Phase 2 指标门禁

系统必须为 StatefulSet Phase 2 输出误动作率、回退率、冻结触发率等关键指标，并作为发布阻断依据。

#### 场景: 指标越线回滚

- **当** 任一 Phase 2 关键指标超过阈值
- **那么** 系统必须触发自动降级或提示立即回滚到只读模式

### 需求:执行链路去硬编码
系统必须将 StatefulSet L2 候选窗口、冻结时长、灰度阈值与降级条件全部配置化，禁止硬编码策略值；同时必须将快照保留窗口、恢复超时、每对象快照上限等参数配置化；Deployment 分层升级阈值与阻断条件也必须配置化。

#### 场景: L2 参数读取
- **当** 系统执行 StatefulSet L2 判定与执行
- **那么** 必须从声明式配置读取全部 L2 控制参数

#### 场景: 快照策略参数读取
- **当** 系统执行快照创建或恢复
- **那么** 必须从声明式配置读取快照策略参数并应用边界约束

#### 场景: Deployment 分层参数读取
- **当** 系统执行 Deployment L1/L2/L3 分层迁移判定
- **那么** 必须从声明式配置读取升级与阻断参数

### 需求:StatefulSet L2 灰度发布控制
系统必须支持 StatefulSet L2 的灰度发布控制，至少包含命名空间白名单、审批开关、快速回退开关与冻结阈值。

#### 场景: L2 灰度范围受控
- **当** 仅对白名单命名空间开启 StatefulSet L2
- **那么** 非白名单命名空间必须继续保持 L1+L3 路径

### 需求:StatefulSet L2 发布阻断指标
系统必须输出 StatefulSet L2 成功率、失败回退率、冻结触发率与降级率，并作为发布阻断依据。

#### 场景: 指标越线阻断
- **当** 任一 L2 关键指标超过阈值
- **那么** 系统必须自动降级并提示回退到保守模式

### 需求:快照容量与时延发布门禁
系统必须输出快照创建成功率、恢复成功率、恢复时延与对象级快照容量占用，并将其作为发布阻断指标。

#### 场景: 快照指标越线阻断
- **当** 快照失败率或恢复时延超过阈值
- **那么** 系统必须阻断继续灰度并提示回退到保守模式

### 需求:Deployment 分层发布门禁

系统必须为 Deployment 分层路径输出 L1 成功率、L2 回滚成功率、L3 降级率与分层阻断率，并将其作为发布阻断指标；当任一指标越线时必须输出统一原因码并执行保守降级动作。

#### 场景: Deployment 分层指标越线阻断

- **当** Deployment 分层关键指标超过阈值
- **那么** 系统必须阻断继续灰度并提示回退到保守模式

#### 场景: Deployment 指标越线触发统一原因码

- **当** Deployment 发布门禁因任一指标越线被阻断
- **那么** 系统必须输出统一原因码、阈值快照与恢复建议

### 需求:生产门禁判定报告输出

系统必须在每次灰度窗口结束后输出门禁判定报告，包含判定结果、越线项、回退动作与恢复条件。

#### 场景: 灰度窗口结束生成报告

- **当** 灰度观测窗口结束
- **那么** 系统必须生成可检索的门禁判定报告

### 需求:交付门禁绑定发布门禁
系统必须将质量门禁结论绑定到发布门禁判定；当质量门禁结论为 `block` 时，发布流程必须停止推进并输出阻断原因。

#### 场景: 质量门禁阻断发布推进
- **当** 统一质量门禁返回 `block`
- **那么** 系统必须拒绝进入后续发布阶段并记录阻断证据

### 需求:发布前质量证据完整性
系统必须在发布窗口结束时输出质量门禁证据摘要，至少包含检查项通过性、失败分类与恢复条件，且该摘要必须可检索。

#### 场景: 发布窗口生成质量证据摘要
- **当** 预发布或发布窗口结束
- **那么** 系统必须生成并持久化质量门禁证据摘要

### 需求:门禁结果映射事故响应等级
系统必须将发布门禁结果映射为事故响应等级，并输出对应值班动作建议；其中 `block` 必须触发最高优先级响应路径。

#### 场景: block 触发最高优先级响应
- **当** 发布门禁结果为 `block`
- **那么** 系统必须输出最高优先级响应等级并要求人工确认后方可继续

### 需求:降级路径恢复条件约束
系统必须在 `degrade` 状态下输出明确恢复条件，未满足恢复条件前禁止自动回到放量路径。

#### 场景: degrade 状态未满足恢复条件
- **当** 系统处于 `degrade` 且恢复条件未满足
- **那么** 系统必须保持保守路径并持续输出恢复差距信息
