## ADDED Requirements

### 需求:多工作负载运行态门禁输入

系统必须在运行态门禁输入中包含工作负载类型与能力级别（read-only / writable），用于统一策略判定。

#### 场景: StatefulSet 门禁输入构建

- **当** 目标工作负载为 StatefulSet
- **那么** 系统必须构建完整门禁输入并标记为 `read-only` 能力级别

### 需求:生产灰度开关

系统必须提供 StatefulSet 只读接入灰度开关，并默认关闭自动写操作能力。

#### 场景: 灰度启用只读接入

- **当** 运维启用 StatefulSet 接入开关
- **那么** 系统必须仅开启只读评估与观测输出，不得开启自动写操作

## MODIFIED Requirements

### 需求:执行链路去硬编码

系统必须禁止运行时硬编码业务阈值与时间窗口；在多工作负载模式下，所有门禁与保守参数必须由声明式配置驱动并可按 workloadKind 扩展。

#### 场景: 按工作负载读取策略

- **当** 系统执行门禁与保守判定
- **那么** 系统必须从请求配置读取阈值并根据 workloadKind 应用对应能力边界

### 需求:观测缺失自保护

系统在部分观测组件不可用时必须保持主流程安全降级；对 StatefulSet 只读路径同样必须输出“观测降级”证据。

#### 场景: StatefulSet 观测降级

- **当** StatefulSet 只读路径发生 Event 或 Metric 上报失败
- **那么** 系统必须继续安全判定并记录可检索降级状态
