## 需求

### 需求:CRD 字段契约完整性

系统必须为对外暴露的 CRD 字段声明默认值、可选性和校验约束（枚举、范围或正则），禁止无约束进入生产发布流程。

#### 场景: 字段缺失约束被阻断

- **当** API 变更引入或修改 CRD 字段但缺少默认值或校验约束
- **那么** 系统必须阻断交付并输出缺失字段清单

### 需求:API 兼容性分类与迁移路径

系统必须对每次 API 变更提供兼容性分类（backward-compatible、migration-required、version-bump-required）并给出迁移路径。

#### 场景: 迁移必需变更缺少迁移路径

- **当** API 变更被标记为 migration-required 但未提供迁移步骤
- **那么** 系统必须阻断发布并提示补全迁移文档

### 需求:状态语义最小集合

系统必须在状态输出中稳定反映策略阶段、最近动作、失败原因和下一步建议，禁止仅输出不可执行的自由文本。

#### 场景: 状态语义字段不完整

- **当** 一次策略执行完成后状态输出缺失任一语义字段
- **那么** 系统必须判定契约不通过并输出缺失项

### 需求:契约证据可审计

系统必须为 API 契约判定生成可检索证据，至少包含变更对象、兼容性分类、约束校验结果和建议动作。

#### 场景: 查询历史契约判定

- **当** 运维查询某次发布的 API 契约判定
- **那么** 系统必须返回完整证据并可关联到对应发布窗口

## ADDED Requirements

### 需求:首发 L1 状态语义最小契约

系统必须为 Deployment 首发 L1 输出稳定的状态语义，至少包含当前阶段、最近动作、失败原因或阻断原因、最近门禁裁决和下一步建议，禁止仅依赖自由文本日志表达结果。

#### 场景: 首发 L1 完成后查询状态

- **当** 运维或测试系统读取一次 Deployment 首发 L1 处理结果
- **那么** 系统必须返回完整且可检索的状态语义字段集合

### 需求:首发 L1 契约与约束一致性

系统必须保证首发 L1 相关 CRD 字段、默认值、校验约束和生成物保持一致，禁止出现实现依赖字段存在但契约未声明的情况。

#### 场景: 首发 L1 引入或调整字段

- **当** 首发 L1 变更涉及新增或调整 CRD 字段
- **那么** 系统必须同步更新默认值、校验约束和生成物，并使契约检查通过
