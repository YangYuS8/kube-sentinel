# local-deployment-and-dev-loop 规范

## ADDED Requirements

### 需求:本地与测试环境必须具备最小可安装入口

系统必须为 Kube-Sentinel 提供一个最小可安装入口，使测试环境能够以统一方式完成 CRD 安装、控制器部署与默认配置加载，禁止仅依赖分散的手工步骤作为首选部署路径。

#### 场景: 新环境执行最小安装

- **当** 开发者或测试人员在一个新的 Kubernetes 测试环境中安装 Kube-Sentinel
- **那么** 系统必须能通过统一入口完成 CRD、控制器与默认配置的安装，并输出可继续执行 smoke 验证的下一步说明

### 需求:本地开发回路必须提供环境自检

系统必须在本地开发回路中显式检查 kube context、CRD 安装状态、关键端口占用和基础集群健康，禁止让操作者在明显未就绪的环境中继续闭环验证。

#### 场景: 环境未准备好

- **当** 开发者执行本地开发入口，但当前 kube context、CRD、端口或集群基础组件未满足前置条件
- **那么** 系统必须阻断继续执行，并输出可诊断的失败分类与修复提示

### 需求:本地开发回路必须支持最小 smoke 验证

系统必须提供面向本地调试的最小 smoke 验证入口，至少覆盖 Deployment 告警接入、HealingRequest 创建、默认安全阻断路径和放宽单次测试请求后的成功闭环路径。

#### 场景: 执行本地 smoke 验证

- **当** 开发者在满足前置条件的本地环境执行 smoke 验证入口
- **那么** 系统必须能够验证默认阻断语义与单次请求成功闭环，并输出对应的对象状态与结果说明

### 需求:minikube 友好约束必须显式文档化

系统必须将 minikube 下的代理前提、基础组件就绪条件和小规模集群 blast radius 限制显式文档化，禁止将这些环境约束仅隐含在故障排查经验中。

#### 场景: 使用 minikube 首次联调

- **当** 开发者首次在 minikube 上执行 Kube-Sentinel 本地联调
- **那么** 文档与入口必须明确说明代理配置原则、集群健康检查步骤和小集群下 blast radius 的测试处理方式

### 需求:本地构建路径与官方预构建镜像路径必须并存

系统必须同时保留“开发者本地构建并导入测试集群”的路径与“直接消费官方预构建镜像”的路径，禁止在引入官方 GHCR 镜像后将本地构建回路视为废弃或默认移除。

#### 场景: 开发者继续使用本地构建回路

- **当** 开发者在本地 minikube 或测试环境中需要快速验证未发布改动
- **那么** 系统必须继续支持本地构建镜像并导入测试集群，而不要求开发者先经过官方镜像发布流程

#### 场景: 测试环境消费官方预构建镜像

- **当** 使用者不需要验证本地未发布改动，而是希望直接部署官方构建产物
- **那么** 系统必须允许其通过显式镜像引用使用官方 GHCR 镜像完成部署

### 需求:远端镜像消费说明必须显式区分稳定版与预发布版

系统必须在本地与测试环境部署说明中明确区分稳定版镜像入口与预发布镜像入口，禁止把 `latest`、预发布标签和本地联调镜像混写为同一类使用建议。

#### 场景: 使用者选择远端镜像部署

- **当** 使用者查看部署说明并选择消费官方远端镜像
- **那么** 系统必须明确说明稳定版标签、预发布标签和本地构建镜像各自适用的场景与边界

### 需求:本地与测试环境必须说明控制台接入前置条件

系统必须在本地和测试环境文档中显式说明 Headlamp 与 Grafana 接入所需的最小前置条件，至少包括 Kubernetes 资源访问方式、Prometheus 抓取前提和适用场景，禁止把控制台接入前提留在零散排障经验中。

#### 场景: 首次在测试环境启用控制台

- **当** 开发者首次在本地或测试集群中启用 Headlamp 与 Grafana
- **那么** 系统必须明确说明对象视图、指标视图各自依赖的前置条件与接入步骤

### 需求:本地开发回路必须区分对象视图验证与指标视图验证

系统必须在本地开发回路中区分对象视图验证与指标视图验证，禁止将 Headlamp、Grafana 和控制器功能验证混写为单一无边界的“UI 检查”步骤。

#### 场景: 执行控制台联调

- **当** 开发者在本地或测试环境中验证控制台集成
- **那么** 系统必须能够分别说明如何验证对象视图可读性与指标视图可用性
