## ADDED Requirements

### 需求:本地与测试环境必须具备最小可安装入口

系统必须为 Kube-Sentinel 提供一个最小可安装入口，使测试环境能够以统一方式完成 CRD 安装、控制器部署与默认配置加载，禁止仅依赖分散的手工步骤作为首选部署路径。

#### 场景: 新环境执行最小安装

- **当** 开发者或测试人员在一个新的 Kubernetes 测试环境中安装 Kube-Sentinel
- **那么** 系统必须能通过统一入口完成 CRD、控制器与默认配置的安装，并输出可继续执行 smoke 验证的下一步说明

### 需求:本地开发回路必须提供环境自检

系统必须在本地开发回路中显式检查 kube context、CRD 安装状态、关键端口占用和基础集群健康，禁止让操作者在明显未就绪的环境中继续闭环验证。

#### 场景: 环境未准备好

- **当** 开发者执行本地开发入口，但当前 kube context、CRD、端口或集群基础组件未满足前置条件
- **那么** 系统必须阻断继续执行，并输出可诊断的失败分类与修复提示

### 需求:本地开发回路必须支持最小 smoke 验证

系统必须提供面向本地调试的最小 smoke 验证入口，至少覆盖 Deployment 告警接入、HealingRequest 创建、默认安全阻断路径和放宽单次测试请求后的成功闭环路径。

#### 场景: 执行本地 smoke 验证

- **当** 开发者在满足前置条件的本地环境执行 smoke 验证入口
- **那么** 系统必须能够验证默认阻断语义与单次请求成功闭环，并输出对应的对象状态与结果说明

### 需求:minikube 友好约束必须显式文档化

系统必须将 minikube 下的代理前提、基础组件就绪条件和小规模集群 blast radius 限制显式文档化，禁止将这些环境约束仅隐含在故障排查经验中。

#### 场景: 使用 minikube 首次联调

- **当** 开发者首次在 minikube 上执行 Kube-Sentinel 本地联调
- **那么** 文档与入口必须明确说明代理配置原则、集群健康检查步骤和小集群下 blast radius 的测试处理方式
