## ADDED Requirements

### 需求:官方控制器镜像必须通过受控版本标签发布

系统必须将 Kube-Sentinel 官方控制器镜像的正式发布入口固定为 `v` 前缀语义化版本 tag push，禁止由普通分支 push、非语义化标签或未声明版本语义的事件直接触发正式镜像发布。

#### 场景: 稳定版标签触发正式发布

- **当** 仓库收到符合 `vX.Y.Z` 格式的 tag push
- **那么** 系统必须启动官方镜像发布流程，并为该版本生成可消费的正式镜像标签

#### 场景: 非正式事件不得触发正式发布

- **当** 触发事件是普通分支 push、非 `v` 前缀标签或不符合语义化版本格式的标签
- **那么** 系统必须禁止将该事件作为正式镜像发布入口

### 需求:官方镜像必须以单一引用提供 amd64 与 arm64 多架构支持

系统必须将官方控制器镜像发布为单一可消费引用，并要求该引用同时包含 `linux/amd64` 与 `linux/arm64` 两种架构的镜像 manifest；禁止为同一正式版本要求使用者手动在两个架构专用镜像名之间做选择。

#### 场景: 发布正式版本时生成多架构镜像

- **当** 系统完成一次正式控制器镜像发布
- **那么** 该镜像引用必须同时包含 `linux/amd64` 与 `linux/arm64` 的可拉取 manifest

#### 场景: 使用者消费官方镜像

- **当** amd64 或 arm64 集群节点拉取同一官方镜像引用
- **那么** 系统必须允许节点自动解析并拉取匹配自身架构的镜像层，而不需要人工改写镜像名称

### 需求:稳定版、预发布版与 latest 标签语义必须明确分离

系统必须将 `latest` 保留为稳定版镜像入口：稳定版 `vX.Y.Z` 发布时必须同时产出版本标签与 `latest`；预发布标签如 `vX.Y.Z-rc.N`、`vX.Y.Z-beta.N` 发布时必须仅产出预发布版本标签，禁止覆盖 `latest`。

#### 场景: 稳定版更新 latest

- **当** 系统处理符合 `vX.Y.Z` 格式的稳定版 tag
- **那么** 系统必须发布该版本标签并更新 `latest`

#### 场景: 预发布不得更新 latest

- **当** 系统处理带预发布后缀的语义化版本 tag
- **那么** 系统必须仅发布对应预发布标签，并禁止更新 `latest`

### 需求:官方镜像分发目标必须固定为 GHCR 根路径

系统必须将官方运行镜像分发到 `ghcr.io/yangyus8/kube-sentinel`，并将其视为当前仓库唯一正式运行镜像名称；禁止在没有新增正式工件的前提下再引入额外的运行镜像路径层级。

#### 场景: 发布官方控制器镜像

- **当** 系统发布 Kube-Sentinel 官方运行镜像
- **那么** 镜像名称必须是 `ghcr.io/yangyus8/kube-sentinel`

### 需求:手动触发只能作为受控运维入口

系统可以提供手动触发的镜像发布工作流用于预发布验证或补发，但该入口必须保留受控运维语义，禁止默认覆盖 `latest`，也禁止绕过版本标签规则成为正式发布事实来源。

#### 场景: 手动触发预发布验证

- **当** 操作者手动触发镜像发布工作流用于预发布验证或补发
- **那么** 系统必须限制其行为为受控运维入口，并禁止默认覆盖 `latest`
