## 需求

### 需求:发布就绪度指标暴露

系统必须暴露发布就绪度相关指标，至少包括发布判定次数、各等级占比、值班模板触发次数与人工覆盖次数。

#### 场景: 查询发布就绪趋势

- **当** 运维查看发布稳定性趋势
- **那么** 系统必须提供可按时间窗口聚合的发布就绪度指标

### 需求:值班告警映射最小集

系统必须定义值班告警最小集，覆盖发布阻断、持续降级、审批缺失与演练证据过期场景。

#### 场景: 演练证据过期告警

- **当** 发布门禁检测到演练证据过期
- **那么** 系统必须触发可检索告警并附带补救建议

### 需求:交付运营指标暴露

系统必须暴露持续交付关键指标，至少包含门禁通过率、阻断率、恢复耗时与演练覆盖率。

#### 场景: 查询交付趋势

- **当** 运维查询交付稳定性
- **那么** 系统必须返回可按时间窗口聚合的交付运营指标

### 需求:周期汇总报告输出

系统必须按固定周期输出交付汇总报告，并包含趋势变化与异常摘要。

#### 场景: 周期汇总生成

- **当** 到达汇总周期
- **那么** 系统必须生成包含趋势对比和异常摘要的交付报告

### 需求:控制台指标必须具备标准抓取入口

系统必须以标准 Prometheus 抓取方式暴露控制器指标，使 Grafana 能够通过 Prometheus 数据源消费 `kube_sentinel_*` 指标，禁止仅依赖临时端口转发或人工拼接查询规则作为唯一接入方式。

#### 场景: 测试环境接入 Grafana

- **当** 测试环境启用 Grafana 面板查看 Kube-Sentinel 运行态指标
- **那么** 系统必须提供标准的指标抓取入口与最小接入配置，使 Prometheus 能稳定抓取控制器指标

### 需求:指标视图必须覆盖最小运维分组

系统必须为 Grafana 等指标视图定义最小运维分组，至少覆盖总体触发与成功率、策略执行结果、快照结果和门禁/发布相关趋势，禁止仅暴露无分组原始指标而不说明其控制台消费语义。

#### 场景: 运维查看最小仪表盘

- **当** 运维打开 Kube-Sentinel 的最小运行态仪表盘
- **那么** 仪表盘必须能够按分组展示总体触发、策略结果、快照结果与门禁趋势四类视图
