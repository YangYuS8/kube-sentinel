## 目的

定义 Deployment 与 StatefulSet 的统一编排入口与执行约束，保证多工作负载场景下的策略分流清晰、执行幂等且可追踪。
## 需求
### 需求:多工作负载统一编排入口

系统必须提供统一编排入口处理 Deployment 与 StatefulSet，并通过能力级别控制具体动作路径。

#### 场景: 按能力级别分流

- **当** 编排器接收到不同工作负载请求
- **那么** 系统必须根据能力级别分流到对应可写/只读路径

### 需求:编排链路幂等执行
系统必须保证多工作负载下同一告警重复触发不会导致重复副作用；在 StatefulSet L2 模式下，必须对同一窗口内重复回滚执行进行幂等阻断；对快照创建也必须使用同一幂等键避免重复持久化。

#### 场景: StatefulSet L2 重复触发
- **当** StatefulSet 在同一幂等窗口重复触发 L2 回滚
- **那么** 系统必须拒绝重复回滚并输出幂等阻断证据

#### 场景: 快照重复创建请求
- **当** 同一工作负载在同一幂等窗口重复触发快照创建
- **那么** 系统必须复用或拒绝重复快照写入并记录原因码

### 需求:StatefulSet L1-L2-L3 阶段化编排
系统必须在统一编排状态机中支持 StatefulSet 的 L1→L2→L3 渐进式处置路径。

#### 场景: L1 失败后升级
- **当** StatefulSet L1 失败且满足升级条件
- **那么** 系统必须升级执行 L2 回滚判定

#### 场景: L2 无候选降级
- **当** StatefulSet L2 未找到健康候选或证据不足
- **那么** 系统必须降级到 L3 人工介入

### 需求:自动动作前持久快照校验
系统在 Deployment 与 StatefulSet 任一自动写操作执行前，必须先创建可持久快照；快照创建失败时必须禁止写操作并降级为只读评估。

#### 场景: 快照创建失败阻断写操作
- **当** 编排器在执行 L1 或 L2 前创建持久快照失败
- **那么** 系统必须阻断本次写操作并记录可诊断失败原因
