## ADDED Requirements

### 需求:对象级熔断
系统必须实现对象级熔断：当同一 Deployment 在窗口期内连续失败达到阈值时，必须暂停该对象后续自动自愈动作。

#### 场景: 单对象连续失败超阈值
- **当** 同一 Deployment 在策略执行中连续失败次数达到配置阈值
- **那么** 系统必须将该 Deployment 标记为对象级熔断状态，并仅保留只读评估与告警

### 需求:域级熔断
系统必须实现域级熔断（命名空间或全局）：当失败密度与影响面超过阈值时，必须暂停该范围的新自愈写操作。

#### 场景: 命名空间出现集中失败
- **当** 某命名空间内在统计窗口期内出现超过阈值的熔断事件或失败密度
- **那么** 系统必须触发域级熔断并拒绝该范围内新的自动写操作请求

### 需求:分级治理优先级
系统必须遵循双层熔断优先级：域级熔断生效时，对象级状态不得绕过域级限制。

#### 场景: 对象级恢复但域级仍熔断
- **当** 某 Deployment 的对象级熔断已恢复但所在域级熔断仍未解除
- **那么** 系统必须继续拒绝该对象自动写操作，直到域级熔断解除

### 需求:熔断恢复条件
系统必须定义可验证的熔断恢复条件（冷却时长、失败率回落、人工确认）并写入审计记录。

#### 场景: 达到恢复条件
- **当** 熔断对象或域满足配置的恢复条件
- **那么** 系统必须解除对应熔断状态，并记录解除原因与生效时间
