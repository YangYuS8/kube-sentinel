## 目的

定义分层熔断与命名空间预算阻断策略，确保在故障放大风险出现时能够优先执行安全阻断，并输出可审计证据。
## 需求
### 需求:命名空间阻断预算按应用数计算

系统必须使用命名空间内不健康工作负载占比计算区域阻断预算，不得以 Pod 数直接替代应用数。

#### 场景: 命名空间占比触发

- **当** `UnhealthyWorkloads / TotalWorkloadsInNamespace` 达到阈值
- **那么** 系统必须进入命名空间级只读阻断

### 需求:最小基数保护

系统必须在命名空间工作负载总量过小时启用最小基数保护，避免比例阈值失真导致误阻断。

#### 场景: 小样本命名空间

- **当** 命名空间工作负载总量低于最小基数阈值
- **那么** 系统必须使用绝对值阈值而非比例阈值判定阻断

### 需求:分级治理优先级
系统必须遵循双层熔断优先级，并且熔断阈值、冷却时间必须由请求配置驱动；当命名空间预算触发时，优先执行区域阻断策略；Deployment 分层动作命中熔断后必须禁止阶段升级；当熔断与发布门禁并发命中时必须输出统一裁决结果。

#### 场景: 配置阈值生效
- **当** 请求设置对象级与域级熔断阈值
- **那么** 系统必须按请求阈值判断是否开启对应层级熔断

#### 场景: 区域阻断优先
- **当** 命名空间级预算与对象级熔断同时命中
- **那么** 系统必须优先执行命名空间级只读阻断

#### 场景: Deployment 熔断阻止升级
- **当** Deployment L1 失败且熔断已打开
- **那么** 系统必须禁止升级到 L2 并输出熔断阻断原因

#### 场景: 并发命中裁决可解释
- **当** 熔断与发布门禁同时命中
- **那么** 系统必须输出裁决优先级与最终阻断动作

### 需求:熔断触发可视化

系统必须在熔断触发时输出熔断层级、触发阈值、当前计数与预计恢复时间；在预算阻断场景必须额外输出占比证据。

#### 场景: 域级熔断触发

- **当** 失败密度达到域级阈值
- **那么** 系统必须记录域级阈值、当前计数与恢复时间并输出可检索事件

#### 场景: 命名空间预算阻断触发

- **当** 命名空间不健康应用占比达到阈值
- **那么** 系统必须输出占比计算结果与阻断持续信息

### 需求:Deployment 分层动作与熔断联动
系统必须将 Deployment L1/L2 分层写动作纳入对象级与域级熔断联动，熔断打开时仅允许只读评估与告警。

#### 场景: Deployment 分层动作命中熔断
- **当** Deployment L1 或 L2 动作命中对象级或域级熔断
- **那么** 系统必须阻断写操作并降级到只读评估

### 需求:门禁与熔断冲突裁决
系统必须在发布门禁与熔断同时命中时执行固定裁决顺序，并输出裁决来源与最终动作。

#### 场景: 熔断与门禁并发命中
- **当** 同一对象同时命中发布门禁越线与熔断打开
- **那么** 系统必须按裁决顺序输出唯一阻断结果
