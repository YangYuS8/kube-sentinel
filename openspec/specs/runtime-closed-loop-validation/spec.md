## ADDED Requirements

### 需求:预生产验证通过前禁止生产灰度

运行时闭环验证必须将预生产验证结果作为生产灰度前置条件，未通过时禁止放量并要求先完成修复验证；用于本地与测试环境的最小 smoke 回路必须可重复执行，并明确区分默认阻断路径与放宽单次测试请求后的成功路径，同时输出真实 phase 和 oncall state。对于 StatefulSet 受控动作与 L2 回滚，系统还必须提供显式启用的真实集群验证入口，未完成该验证时不得宣称核心工作负载闭环已经成立。

#### 场景: 预生产验证失败

- **当** 闭环验证输出失败或证据不完整
- **那么** 系统必须阻断生产灰度并输出修复建议

#### 场景: 预生产验证通过

- **当** 闭环验证输出通过且证据完整
- **那么** 系统必须允许进入生产灰度放量判定

#### 场景: 本地 smoke 可重复执行

- **当** 开发者在同一测试环境中重复执行本地 smoke 验证
- **那么** 系统必须能够稳定复现默认阻断路径与成功闭环路径，并输出一致的前置条件、phase 和 oncall state 说明

### 需求:闭环演练必须验证 oncall state
系统必须让本地与测试环境的闭环演练同时验证 runtime phase 和 oncall state，禁止仅凭内部 phase 断言就宣称值班体验正确。

#### 场景: 本地 smoke 验证值班语义
- **当** 开发者在本地或测试环境执行夜间值班闭环演练
- **那么** 演练结果必须同时说明真实 phase 和对外 oncall state

#### 场景: StatefulSet 真实性验证未完成

- **当** StatefulSet 仍只有单元测试或 fake 环境验证，而未完成显式启用的真实集群闭环验证
- **那么** 系统必须把该能力标记为待验证，不得作为核心自愈对象闭环已完成的证据

### 需求:StatefulSet 真实性验证入口

系统必须提供可重复执行的 StatefulSet 真实性验证入口，用于在真实 Kubernetes 集群中验证受控动作、L2 回滚、冻结和 snapshot restore 联动；该入口必须显式启用，并在不满足前置条件时清晰跳过。

#### 场景: 显式启用真实集群验证

- **当** 开发者在具备可用 kube context 与测试前置的环境中显式启用 StatefulSet 真实性验证
- **那么** 系统必须执行真实集群验证并输出候选选择、回滚结果和冻结/恢复证据

#### 场景: 前置条件不足时跳过

- **当** 未显式启用真实集群验证，或测试环境缺少必需前置条件
- **那么** 系统必须清晰跳过该验证，而不是产生误导性的失败结果

### 需求:回滚演练达标阈值

运行时闭环验证必须定义回滚演练达标阈值，未达阈值时必须阻断 go-live。

#### 场景: 回滚演练未达标

- **当** 回滚演练成功率或恢复时延未达阈值
- **那么** 系统必须输出 block 并要求先完成补充演练

#### 场景: 回滚演练达标

- **当** 回滚演练满足成功率与恢复时延阈值
- **那么** 系统必须允许该项通过并进入下一闸门
