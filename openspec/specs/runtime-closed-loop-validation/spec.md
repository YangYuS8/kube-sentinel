## ADDED Requirements

### 需求:运行时闭环验收基线
系统必须提供可重复执行的闭环验收流程，覆盖事件输入、策略执行、状态回写与观测输出。

#### 场景: 执行闭环演练
- **当** 运维触发一次标准故障演练
- **那么** 系统必须输出完整链路证据（status、event、metric、audit）

### 需求:Deployment 写操作边界验证
系统必须验证仅 Deployment 可触发自动写操作，非支持类型仅允许只读评估与告警。

#### 场景: 注入非 Deployment 目标
- **当** 演练事件目标为非 Deployment
- **那么** 系统必须拒绝写操作并保留拒绝证据

### 需求:无健康 Revision 降级验证
系统必须验证在无健康 Revision 证据时自动降级到 L3 人工介入。

#### 场景: 健康候选为空
- **当** 回滚候选遍历后无健康 Revision
- **那么** 系统必须将阶段置为 L3 并输出高优先级告警

### 需求:双层熔断有效性验证
系统必须验证对象级与域级熔断均可阻断后续自动写操作。

#### 场景: 熔断阈值触发
- **当** 连续失败达到对象级或域级阈值
- **那么** 系统必须拒绝后续自动写操作并记录熔断级别与恢复条件
