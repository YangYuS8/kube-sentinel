## 目的

定义运行时闭环在 Phase 2 场景下的验收矩阵与回归门禁，确保默认只读、授权可写、失败回退三态行为可重复验证并能阻断风险交付。
## 需求
### 需求:StatefulSet Phase 2 验收矩阵

系统必须提供 StatefulSet Phase 2 的验收矩阵，覆盖授权通过、授权失败、动作失败回退、冻结阻断与回滚恢复场景。

#### 场景: Phase 2 全路径验收

- **当** 执行 Phase 2 演练
- **那么** 系统必须输出各场景通过结果与证据链完整性

### 需求:受控动作回归门禁

系统必须将 StatefulSet Phase 2 纳入 CI 验证，确保核心失败路径与边界值测试稳定通过。

#### 场景: CI 验证失败阻断

- **当** Phase 2 失败路径或边界测试未通过
- **那么** 系统必须阻断交付

### 需求:运行时闭环验收基线

系统必须在既有闭环基线上增加 StatefulSet L2 路径断言，并验证 L1→L2→L3 状态转换可复现；同时必须校验快照创建、回滚失败恢复与幂等阻断证据可复现；Deployment 分层路径必须纳入同等断言。

#### 场景: 三阶段状态断言

- **当** 执行闭环验收脚本
- **那么** 系统必须验证 StatefulSet 三阶段转换符合策略定义

#### 场景: 快照证据断言

- **当** 执行闭环验收脚本覆盖失败路径
- **那么** 系统必须验证快照与恢复证据链完整可检索

#### 场景: Deployment 分层断言

- **当** 执行 Deployment 分层闭环演练
- **那么** 系统必须验证 Deployment L1/L2/L3 迁移与阻断语义符合策略定义

### 需求:StatefulSet Phase 3 验收矩阵
系统必须提供 StatefulSet Phase 3 验收矩阵，覆盖 L1 失败转 L2、L2 成功、L2 失败恢复、冻结阻断与 L3 降级。

#### 场景: Phase 3 全路径验收
- **当** 执行 StatefulSet Phase 3 演练
- **那么** 系统必须输出各路径通过结果与证据链完整性

### 需求:StatefulSet L2 回归门禁
系统必须将 StatefulSet L2 纳入 CI 回归门禁，且失败路径与边界值测试未通过时必须阻断交付。

#### 场景: L2 失败路径阻断
- **当** L2 回滚失败恢复或冻结路径测试未通过
- **那么** 系统必须阻断交付

### 需求:快照失败路径验收
系统必须将“快照创建失败阻断写操作”“恢复失败进入冻结与人工介入”纳入闭环验收矩阵。

#### 场景: 快照创建失败阻断
- **当** 演练注入快照创建失败
- **那么** 系统必须阻断自动写操作并输出只读评估告警

#### 场景: 恢复失败升级人工介入
- **当** 演练注入恢复执行失败
- **那么** 系统必须进入冻结并输出 L3 人工介入建议

### 需求:Deployment 三阶段验收矩阵
系统必须提供 Deployment L1/L2/L3 三阶段验收矩阵，覆盖 L1 成功、L1 失败升级 L2、L2 失败降级 L3 与恢复路径；并必须增加灰度门禁阻断与自动降级断言。

#### 场景: Deployment 三阶段全路径验收
- **当** 执行 Deployment 分层闭环演练
- **那么** 系统必须输出三阶段路径通过结果与证据链完整性

#### 场景: Deployment 门禁阻断断言
- **当** 执行 Deployment 灰度门禁演练
- **那么** 系统必须验证门禁阻断原因码与降级动作语义一致

### 需求:门禁阻断与降级演练矩阵
系统必须在闭环演练中覆盖 `allow`、`block`、`degrade` 三类门禁结果，并验证每类结果的证据输出完整性。

#### 场景: 三类门禁结果全覆盖
- **当** 执行预生产门禁演练
- **那么** 系统必须输出 allow/block/degrade 三类路径的断言结果

### 需求:质量门禁纳入闭环验收矩阵
系统必须将统一质量门禁纳入闭环验收矩阵，并覆盖通过、阻断、受控降级三类结果断言。

#### 场景: 闭环演练覆盖三类门禁结果
- **当** 执行运行时闭环演练
- **那么** 系统必须输出 `allow`、`block`、`degrade` 三类门禁断言结果

### 需求:预提交与 CI 门禁一致性验证
系统必须验证预提交检查与 CI 检查在核心门禁项上的结果语义一致；当语义不一致时必须判定为验收失败。

#### 场景: 预提交与 CI 结果不一致
- **当** 同一提交在预提交与 CI 的核心门禁结论不一致
- **那么** 系统必须将该差异标记为失败并阻断验收

### 需求:失败路径可诊断性断言
系统必须在闭环验收中校验门禁失败输出包含失败类别、根因提示与修复建议；缺失任一字段时必须判定为不通过。

#### 场景: 门禁失败输出字段缺失
- **当** 演练触发门禁失败且诊断字段不完整
- **那么** 系统必须判定闭环验收失败

### 需求:SLO 越线到响应升级闭环断言
系统必须在闭环演练中验证 SLO 越线后从门禁判定到响应升级的完整链路。

#### 场景: 越线触发升级链路
- **当** 演练注入 SLO 越线条件
- **那么** 系统必须输出门禁结果、升级等级与建议动作的完整证据

### 需求:恢复前置条件闭环验证
系统必须在闭环演练中断言恢复前置条件的满足状态；未满足时不得标记恢复完成。

#### 场景: 恢复条件未满足
- **当** 演练中恢复前置条件缺失
- **那么** 系统必须保持降级或阻断状态并报告缺失项

### 需求:值班可诊断性输出断言
系统必须验证值班输出至少包含门禁类别、根因提示、修复建议与推荐 runbook 标识。

#### 场景: 值班输出字段不完整
- **当** 演练生成响应建议但缺少关键字段
- **那么** 系统必须判定闭环验收失败

### 需求:灰度放量与回退演练断言

系统必须在闭环演练中验证灰度放量步骤、回退触发条件与回退执行证据。

#### 场景: 灰度阶段触发回退

- **当** 演练注入预算越线导致回退条件命中
- **那么** 系统必须执行回退并输出阶段切换证据

### 需求:阈值调优闭环断言

系统必须验证阈值调优流程包含审批、观察窗口和回滚路径，缺失任一环节即判定失败。

#### 场景: 调优流程缺少审批

- **当** 演练执行阈值调优但缺少审批记录
- **那么** 系统必须判定演练失败并阻断调优生效

### 需求:复盘动作可审计断言

系统必须验证复盘输出至少包含越线原因、处置动作、阈值调整结论与后续观察计划。

#### 场景: 复盘字段缺失

- **当** 演练输出复盘报告但缺少关键字段
- **那么** 系统必须判定闭环验收失败

### 需求:闭环演练值班动作完整性断言

系统必须在闭环演练中校验值班动作模板完整性，至少校验告警级别、runbook、审批触发点与回滚建议。

#### 场景: 演练输出模板字段缺失

- **当** 演练结果缺失任一值班动作关键字段
- **那么** 系统必须判定演练失败并阻断验收

### 需求:演练汇总报告一致性断言

系统必须验证演练汇总报告与门禁判定摘要一致；若摘要结论与明细证据冲突必须判定失败。

#### 场景: 摘要与明细冲突

- **当** 汇总报告摘要结论与明细证据不一致
- **那么** 系统必须阻断发布并输出冲突项

### 需求:预生产 dry-run 自动执行

系统必须在交付流水线中自动执行预生产 dry-run，并覆盖关键运行路径验证。

#### 场景: 触发预生产 dry-run

- **当** 发布前流水线进入验证阶段
- **那么** 系统必须自动执行 dry-run 并返回结构化结果

### 需求:三态门禁覆盖断言

dry-run 必须覆盖 allow/degrade/block 三类门禁语义，任一缺失时必须判定验证失败。

#### 场景: 缺失门禁路径覆盖

- **当** dry-run 结果未覆盖 allow/degrade/block 中任意一类
- **那么** 系统必须标记验证失败并阻断发布推进
