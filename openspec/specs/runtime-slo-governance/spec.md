## 目的

定义运行时 SLO 治理契约，统一约束目标指标、预算窗口、越线动作与审计证据，确保发布与运行行为在同一稳定性语义下执行。
## 需求
### 需求:运行时 SLO 治理契约

系统必须定义运行时 SLO 治理契约，至少包含目标指标、评估窗口、失败预算与越线动作等级。

#### 场景: SLO 契约可被统一读取

- **当** 系统进入发布前或运行中评估阶段
- **那么** 系统必须按统一契约读取并执行 SLO 评估逻辑

### 需求:SLO 越线动作分级

系统必须将 SLO 评估结果映射为 `allow`、`degrade`、`block` 三类动作，并且每类动作必须定义发布行为与响应要求。

#### 场景: SLO 结果映射为三态动作

- **当** SLO 评估完成
- **那么** 系统必须输出唯一动作等级并应用对应控制语义

### 需求:失败预算与趋势判定

系统必须基于窗口化失败预算进行越线判定，禁止仅根据单次事件直接触发长期阻断。

#### 场景: 窗口内预算消耗触发越线

- **当** 评估窗口内预算消耗达到阈值
- **那么** 系统必须按策略触发降级或阻断并记录触发依据

### 需求:治理证据可审计

系统必须输出 SLO 评估证据，至少包含评估时间窗、预算消耗、动作等级与恢复前置条件。

#### 场景: 生成 SLO 评估证据

- **当** 任一次 SLO 评估完成
- **那么** 系统必须生成可检索且可复盘的治理证据

### 需求:分层灰度启用策略
系统必须支持按环境或命名空间分层启用 SLO 治理策略，并且每一层都必须定义进入与回退条件。

#### 场景: 分层灰度启用
- **当** 运维推进下一层 SLO 治理启用
- **那么** 系统必须校验上一层稳定窗口与回退条件状态

### 需求:阈值调优审批与回滚
系统必须为 SLO 阈值调优提供审批记录与回滚能力，未完成审批时禁止生效。

#### 场景: 未审批阈值变更
- **当** 变更请求缺少审批记录
- **那么** 系统必须拒绝阈值更新并保持当前策略

### 需求:观察窗口约束
系统必须在固定观察窗口内评估调优效果；观察窗口未结束前禁止再次修改同一对象阈值。

#### 场景: 观察窗口内重复调优
- **当** 同一对象在观察窗口内再次提交阈值调整
- **那么** 系统必须阻断本次调整并提示剩余等待时间
