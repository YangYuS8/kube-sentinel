## ADDED Requirements

### 需求:Webhook 事件接入
系统必须接收 Alertmanager Webhook 事件，并将告警内容映射为可追踪的自愈请求上下文。

#### 场景: 接收到合法告警事件
- **当** Webhook 服务收到结构合法且目标资源可识别的告警
- **那么** 系统必须创建或更新对应 HealingRequest，并写入触发源与关联键

### 需求:非法或不完整事件处理
系统必须拒绝不完整或非法告警事件，禁止触发任何写操作。

#### 场景: 关键字段缺失
- **当** 告警事件缺少目标资源标识或策略必要字段
- **那么** 系统必须返回可诊断错误并记录拒绝原因

### 需求:事件幂等去重
系统必须对重复告警事件执行幂等去重，避免重复触发冲突动作。

#### 场景: 重复事件短时间重放
- **当** 同一告警在幂等窗口内重复到达
- **那么** 系统必须复用既有请求上下文并跳过重复写操作
