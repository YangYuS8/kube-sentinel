## 需求
### 需求:Deployment L1 最小影响动作
系统必须将 Deployment L1 定义为最小影响自动处置动作，并在动作前完成快照校验。

#### 场景: L1 动作成功
- **当** Deployment L1 动作执行成功
- **那么** 系统必须结束自动处置并进入持续观测

### 需求:Deployment L2 健康版本回滚
系统必须在 L1 失败后进入 L2 候选判定，并仅允许回滚到满足健康与依赖条件的版本。

#### 场景: L1 失败后进入 L2
- **当** Deployment L1 动作失败且满足升级条件
- **那么** 系统必须进入 L2 健康版本回滚流程

#### 场景: L2 候选不足降级
- **当** Deployment 不存在满足条件的健康候选
- **那么** 系统必须降级为 L3 人工介入

### 需求:Deployment L3 人工介入建议
系统必须在无法安全自动处置时进入 L3，并输出结构化人工介入建议。

#### 场景: 自动处置不可证明安全
- **当** Deployment 路径命中证据不足或安全门禁阻断
- **那么** 系统必须进入 L3 并给出下一步建议

### 需求:Deployment 分层审计可解释
系统必须为 Deployment 每次阶段迁移记录触发条件、失败证据、执行结果与下一步建议。

#### 场景: 阶段迁移审计输出
- **当** Deployment 发生阶段迁移
- **那么** 系统必须输出可检索的阶段审计记录
