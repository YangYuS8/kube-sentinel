# statefulset-controlled-healing Delta

## MODIFIED Requirements

### 需求:StatefulSet 四重授权门禁

系统执行 StatefulSet 自动动作前，必须同时满足配置开关、命名空间白名单、对象级审批标记与运行态证据完整性；当真实性验证入口被启用时，系统还必须输出与真实集群验证相匹配的阻断证据，禁止用 fake 环境结论替代真实集群判定。

#### 场景: 任一授权条件不满足

- **当** 四重授权门禁中的任一项失败
- **那么** 系统必须禁止写操作并回退到只读评估

#### 场景: 真实性验证发现授权证据不完整

- **当** StatefulSet 真实性验证入口已启用，但运行态授权证据、对象标记或前置条件与真实集群要求不一致
- **那么** 系统必须阻断真实写动作并输出可检索的失败证据，而不是继续执行受控动作

### 需求:失败回退只读

系统在 StatefulSet 自动动作失败后，必须立即回退到只读模式并冻结后续自动重试，直到人工解锁；当 L2 回滚失败时同样必须执行该规则；若恢复失败，必须记录恢复失败原因码并维持冻结状态。该语义必须能够通过真实集群验证得到重复证明。

#### 场景: L2 失败回退只读

- **当** StatefulSet L2 回滚执行失败
- **那么** 系统必须进入只读冻结状态并要求人工介入

#### 场景: 恢复失败维持冻结

- **当** StatefulSet L2 回滚失败后快照恢复也失败
- **那么** 系统必须维持冻结并输出不可自动重试的诊断结论

#### 场景: 真实性验证中重复触发仍保持冻结

- **当** StatefulSet 在真实集群中已经处于失败冻结窗口，且再次收到同一对象的自动动作请求
- **那么** 系统必须继续阻断自动写操作，并输出剩余冻结或人工解锁要求
