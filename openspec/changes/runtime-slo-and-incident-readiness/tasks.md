## 1. SLO 治理模型与配置落地

- [ ] 1.1 定义运行时 SLO 配置模型（目标指标、评估窗口、预算阈值、动作分级）并接入可声明式读取路径；验收标准：默认配置可被稳定加载，缺省值与边界值输入都有明确行为。
- [ ] 1.2 为 SLO 配置模型补充单元测试与边界测试（空配置、非法阈值、极小/极大窗口）；验收标准：非法配置被阻断，合法配置通过并可产出一致解析结果。
- [ ] 1.3 如涉及 API 字段扩展，同步更新 Helm `values.yaml` 与 `values.schema.json` 约束；验收标准：API/CRD/Helm 一致性检查通过。

## 2. 门禁结果与事故响应联动

- [ ] 2.1 在质量门禁输出中增加 SLO 关联字段（动作等级、预算状态、建议响应级别）；验收标准：每次门禁完成都输出可解析字段，失败路径包含根因与修复建议。
- [ ] 2.2 建立 `allow`/`degrade`/`block` 到事故响应等级与恢复条件的映射逻辑；验收标准：三态映射唯一且可复现，`block` 必须触发阻断与人工确认语义。
- [ ] 2.3 为映射逻辑补充幂等与回滚测试（重复执行、状态抖动、恢复条件未满足）；验收标准：重复输入结果稳定，恢复条件不足时不会错误放量。

## 3. 闭环验收与演练增强

- [ ] 3.1 扩展闭环演练脚本以覆盖“SLO 越线 -> 升级 -> 降级/阻断 -> 恢复条件检查”全路径；验收标准：演练输出包含完整证据链并可自动断言。
- [ ] 3.2 增加“预提交与 CI 门禁语义 + SLO 治理语义”一致性断言；验收标准：任一语义不一致即判定失败并阻断验收。
- [ ] 3.3 为闭环输出增加值班可诊断字段校验（类别、根因、修复建议、runbook 标识）；验收标准：缺失关键字段时演练失败。
