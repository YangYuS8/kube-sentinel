## ADDED Requirements

### 需求:Deployment L1-L2-L3 阶段化编排
系统必须为 Deployment 提供显式 L1→L2→L3 渐进式处置状态机，且每次阶段迁移必须记录触发原因与证据。

#### 场景: L1 失败后升级 L2
- **当** Deployment L1 最小影响动作失败且门禁允许升级
- **那么** 系统必须进入 L2 回滚判定并记录升级依据

#### 场景: L2 证据不足降级 L3
- **当** Deployment L2 候选不足或依赖校验失败
- **那么** 系统必须降级到 L3 人工介入并输出建议

## MODIFIED Requirements

### 需求:编排链路幂等执行
系统必须保证多工作负载下同一告警重复触发不会导致重复副作用；在 StatefulSet L2 模式下，必须对同一窗口内重复回滚执行进行幂等阻断；对快照创建也必须使用同一幂等键避免重复持久化；在 Deployment 分层模式下，必须阻断同一窗口内重复阶段写动作。

#### 场景: StatefulSet L2 重复触发
- **当** StatefulSet 在同一幂等窗口重复触发 L2 回滚
- **那么** 系统必须拒绝重复回滚并输出幂等阻断证据

#### 场景: 快照重复创建请求
- **当** 同一工作负载在同一幂等窗口重复触发快照创建
- **那么** 系统必须复用或拒绝重复快照写入并记录原因码

#### 场景: Deployment 分层重复触发
- **当** Deployment 在同一幂等窗口重复触发同一阶段写动作
- **那么** 系统必须阻断重复执行并输出阶段阻断证据
