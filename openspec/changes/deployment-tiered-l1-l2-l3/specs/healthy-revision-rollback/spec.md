## ADDED Requirements

### 需求:Deployment L2 候选筛选与恢复证据
系统必须在 Deployment L2 回滚前完成候选筛选与依赖校验，并在回滚失败时输出快照恢复证据链。

#### 场景: Deployment L2 回滚失败触发恢复
- **当** Deployment L2 回滚执行失败
- **那么** 系统必须基于快照执行恢复并记录恢复结果

## MODIFIED Requirements

### 需求:回滚证据可观测输出
系统必须在每次 L2 回滚决策中输出候选筛选证据，并明确记录回滚执行动作与失败恢复结果；证据不足时必须明确说明降级原因；同时必须记录对应快照标识与恢复耗时；Deployment 分层路径必须补充阶段升级证据。

#### 场景: StatefulSet 证据不足降级
- **当** StatefulSet 无法建立完整回滚证据链
- **那么** 系统必须禁止 L2 并记录降级到 L3 的依据

#### 场景: 回滚恢复证据可追溯
- **当** 任一工作负载发生 L2 回滚失败并触发恢复
- **那么** 系统必须输出可关联到快照标识的恢复证据链

#### 场景: Deployment 分层升级证据
- **当** Deployment 从 L1 升级到 L2 或降级到 L3
- **那么** 系统必须输出对应升级/降级证据与原因码
