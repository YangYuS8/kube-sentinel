## ADDED Requirements

### 需求:持续交付流水线闭环执行

系统必须在每次合并前自动执行“质量门禁、预生产 dry-run、证据归档”三段式流程；任一阶段失败时必须阻断发布推进。

#### 场景: 任一阶段失败时阻断

- **当** 质量门禁、dry-run 或证据归档任一阶段返回失败
- **那么** 流水线必须输出阻断结论并终止发布推进

### 需求:门禁与演练语义一致性

系统必须保证 allow/degrade/block 门禁语义在质量门禁输出、dry-run 结果和值班动作模板之间保持一致，不一致时必须判定失败。

#### 场景: 语义冲突时失败

- **当** 门禁结论与 dry-run 或值班模板映射不一致
- **那么** 系统必须输出冲突项并返回失败

### 需求:周期演练持续验证

系统必须按固定周期自动执行演练，并至少覆盖 allow/degrade/block 三类门禁路径，同时输出可追踪的趋势结果。

#### 场景: 周期演练产出趋势结果

- **当** 到达设定演练周期
- **那么** 系统必须执行演练并输出包含三类路径覆盖情况的趋势报告
