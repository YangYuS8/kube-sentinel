## 1. 交付流水线基础闭环

- [ ] 1.1 在 CI 工作流中增加“质量门禁 -> 预生产 dry-run -> 证据归档”三段式执行顺序，并定义失败即阻断。
- [ ] 1.2 增加流水线单元测试/脚本测试：覆盖任一阶段失败时必须终止后续阶段并返回非零退出码。
- [ ] 1.3 增加验收用例：输入为三段式全成功/部分失败，预期分别为 allow 或 block，并输出明确失败阶段。

## 2. 质量门禁证据标准化

- [ ] 2.1 扩展质量门禁输出，统一生成机器可读证据文件与人可读摘要，至少包含结论、分类、原因码、修复提示。
- [ ] 2.2 增加脚本测试：覆盖证据字段缺失、格式非法、字段完整三类路径。
- [ ] 2.3 增加验收用例：证据缺失关键字段时必须阻断，证据完整时允许进入下一阶段。

## 3. 预生产 dry-run 自动化与三态覆盖

- [ ] 3.1 实现预生产 dry-run 自动执行入口，并回传结构化结果到流水线。
- [ ] 3.2 增加测试：必须覆盖 allow/degrade/block 三类门禁语义，任一缺失判定失败。
- [ ] 3.3 增加验收用例：给定三类模拟输入，预期 dry-run 结果与门禁语义一致且可追踪。

## 4. 发布判定与值班模板一致性

- [ ] 4.1 增加一致性校验：发布判定结论必须与值班模板等级、runbook、审批触发点一致。
- [ ] 4.2 增加测试：覆盖一致、冲突、未知判定三类路径，冲突必须阻断。
- [ ] 4.3 增加验收用例：冲突时输出冲突项与修复建议，不允许继续发布。

## 5. 人工覆盖审计链路

- [ ] 5.1 扩展审计输出字段：operator override 的人、前后判定、原因、时间戳及关联键。
- [ ] 5.2 增加测试：覆盖覆盖字段完整、字段缺失、重复覆盖幂等性场景。
- [ ] 5.3 增加验收用例：人工覆盖发生后可通过关联键检索完整审计证据。

## 6. 周期演练与交付趋势指标

- [ ] 6.1 增加定时演练任务（nightly 或工作日窗口可配置），并输出周期汇总报告。
- [ ] 6.2 增加指标聚合：门禁通过率、阻断率、恢复耗时、演练覆盖率。
- [ ] 6.3 增加测试：覆盖时间窗口边界、空样本窗口、异常数据输入。

## 7. 配置、契约与发布收口
