## 为什么

当前系统已经具备较完整的自愈策略、质量门禁与 API 契约能力，但发布决策与值班处置仍偏“人工拼接”，缺少统一的发布就绪判定摘要、告警动作模板和闭环演练自动汇总。随着交付频率提升，这会导致值班响应质量波动并增加发布窗口风险。

## 变更内容

- 新增发布就绪与值班自动化能力：定义统一发布判定摘要（allow/degrade/block）与关键证据字段。
- 在运行时闭环验收中增加“值班动作模板化”约束：告警级别、runbook、审批触发点和回滚动作必须可自动映射。
- 在交付质量门禁中补充“演练汇总证据”要求：每次门禁必须输出可检索的发布就绪报告。
- 更新可观测性规范，补充发布就绪度相关指标与最小告警集映射。
- 明确文档与流程收口要求，保证值班人员可独立完成一次阻断处置与恢复评估。

## 功能 (Capabilities)

### 新增功能
- `release-readiness-and-oncall-automation`: 定义发布就绪判定、值班动作模板与演练汇总自动化的规范能力。

### 修改功能
- `runtime-closed-loop-validation`: 增加值班动作映射完整性与复盘自动汇总断言要求。
- `delivery-quality-gates`: 增加发布就绪报告与演练汇总证据输出要求。
- `runtime-production-gating`: 增加发布判定摘要字段与审批触发点约束。
- `observability`: 增加发布就绪度与值班处置效率相关指标/告警要求。

## 影响

- OpenSpec：新增 `release-readiness-and-oncall-automation` 能力规范，并同步更新相关既有能力规范。
- 交付流程：质量门禁输出将新增发布就绪摘要与演练汇总证据。
- 运行时诊断：值班动作将从“依赖经验”转为“模板化、可审计、可回放”。
- 文档/运维：发布与回滚手册将对齐自动化摘要字段，降低值班认知负担。
