## ADDED Requirements

### 需求:本地构建路径与官方预构建镜像路径必须并存

系统必须同时保留“开发者本地构建并导入测试集群”的路径与“直接消费官方预构建镜像”的路径，禁止在引入官方 GHCR 镜像后将本地构建回路视为废弃或默认移除。

#### 场景: 开发者继续使用本地构建回路

- **当** 开发者在本地 minikube 或测试环境中需要快速验证未发布改动
- **那么** 系统必须继续支持本地构建镜像并导入测试集群，而不要求开发者先经过官方镜像发布流程

#### 场景: 测试环境消费官方预构建镜像

- **当** 使用者不需要验证本地未发布改动，而是希望直接部署官方构建产物
- **那么** 系统必须允许其通过显式镜像引用使用官方 GHCR 镜像完成部署

### 需求:远端镜像消费说明必须显式区分稳定版与预发布版

系统必须在本地与测试环境部署说明中明确区分稳定版镜像入口与预发布镜像入口，禁止把 `latest`、预发布标签和本地联调镜像混写为同一类使用建议。

#### 场景: 使用者选择远端镜像部署

- **当** 使用者查看部署说明并选择消费官方远端镜像
- **那么** 系统必须明确说明稳定版标签、预发布标签和本地构建镜像各自适用的场景与边界
