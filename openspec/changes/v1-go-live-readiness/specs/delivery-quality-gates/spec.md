## ADDED Requirements

### 需求:go-live 闸门联动输出
交付质量门禁必须输出 go-live 判定所需字段，至少包含闸门结论、失败类别、修复建议与放量建议。

#### 场景: 门禁阻断 go-live
- **当** 任一质量检查失败或关键证据不完整
- **那么** 门禁必须输出 block 并拒绝进入 go-live 放量阶段

#### 场景: 门禁允许 go-live
- **当** 质量检查全部通过且证据完整
- **那么** 门禁必须输出 allow 并附带 go-live 决策字段

### 需求:release decision pack 机器可读输出
交付质量门禁必须输出机器可读的 release decision pack 文件，并保证字段稳定可解析。

#### 场景: 字段稳定输出
- **当** 连续多次执行 go-live 门禁
- **那么** 系统必须保持决策包字段命名与语义一致

#### 场景: 决策包解析失败
- **当** 下游系统无法解析决策包字段
- **那么** 门禁必须返回 block 并提示字段修复路径
