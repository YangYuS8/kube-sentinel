## ADDED Requirements

### 需求:值班审批与 go-live 判定一致性
系统必须保证值班审批状态与 go-live 判定一致，审批缺失或等级不匹配时必须阻断放量。

#### 场景: 审批缺失
- **当** go-live 判定需要审批但值班审批状态缺失
- **那么** 系统必须输出 block 并提示补齐审批

#### 场景: 审批等级不匹配
- **当** 判定等级与值班审批等级不一致
- **那么** 系统必须阻断放量并输出冲突项

### 需求:人工覆盖与冻结窗口约束
系统必须在冻结窗口与人工覆盖场景下执行强约束：冻结窗口内禁止覆盖放量，覆盖放量必须记录完整审计字段。

#### 场景: 冻结窗口内尝试覆盖
- **当** 当前时间命中冻结窗口且请求人工覆盖
- **那么** 系统必须拒绝覆盖并输出冻结阻断原因

#### 场景: 覆盖审计字段缺失
- **当** 人工覆盖请求缺少覆盖人、原因、时间戳或关联键
- **那么** 系统必须阻断覆盖并提示补齐审计字段
