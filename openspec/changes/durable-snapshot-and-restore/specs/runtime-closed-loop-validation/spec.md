## ADDED Requirements

### 需求:快照失败路径验收
系统必须将“快照创建失败阻断写操作”“恢复失败进入冻结与人工介入”纳入闭环验收矩阵。

#### 场景: 快照创建失败阻断
- **当** 演练注入快照创建失败
- **那么** 系统必须阻断自动写操作并输出只读评估告警

#### 场景: 恢复失败升级人工介入
- **当** 演练注入恢复执行失败
- **那么** 系统必须进入冻结并输出 L3 人工介入建议

## MODIFIED Requirements

### 需求:运行时闭环验收基线
系统必须在既有闭环基线上增加 StatefulSet L2 路径断言，并验证 L1→L2→L3 状态转换可复现；同时必须校验快照创建、回滚失败恢复与幂等阻断证据可复现。

#### 场景: 三阶段状态断言
- **当** 执行闭环验收脚本
- **那么** 系统必须验证 StatefulSet 三阶段转换符合策略定义

#### 场景: 快照证据断言
- **当** 执行闭环验收脚本覆盖失败路径
- **那么** 系统必须验证快照与恢复证据链完整可检索
