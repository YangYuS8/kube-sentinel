## ADDED Requirements

### 需求:PendingVerify 与 Suppressed 状态语义

系统必须在自动动作前进入 `PendingVerify` 观察状态；观察期内恢复的事件必须标记为 `Suppressed`。

#### 场景: 观察期抑制

- **当** 事件在观察期内恢复
- **那么** 系统必须结束当前链路并将结果记录为 `Suppressed`

## MODIFIED Requirements

### 需求:编排链路幂等执行

系统必须保证同一告警在重复触发时不会导致重复副作用，必须以目标对象当前状态作为判定依据，并使用请求声明的幂等窗口执行重复抑制；在保守模式下，观察期未结束前禁止进入写操作。

#### 场景: 自定义幂等窗口生效

- **当** 请求显式配置 `idempotencyWindowMinutes`
- **那么** 系统必须按该窗口判定重复事件并禁止重复写操作

#### 场景: 观察期内阻断写操作

- **当** 请求处于 `PendingVerify` 观察期
- **那么** 系统必须仅执行只读评估并禁止写操作

### 需求:v1alpha1 Deployment 自愈编排范围

系统必须在 v1alpha1 阶段仅对 Deployment 执行自动自愈动作，并且门禁判定必须基于运行态事实输入；当事实输入不可用时必须转入只读阻断。

#### 场景: 运行态门禁输入

- **当** 编排链路执行维护窗口、速率限制与爆炸半径判定
- **那么** 系统必须使用实时动作计数与规模数据，不得使用固定示例值

#### 场景: 门禁输入缺失

- **当** 运行态事实输入构建失败
- **那么** 系统必须只读阻断并输出可检索失败证据
