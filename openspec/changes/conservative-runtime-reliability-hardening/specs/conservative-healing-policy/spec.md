## ADDED Requirements

### 需求:保守执行总原则

系统必须以“绝不误动作”为第一优先级；当安全性无法被证明时，必须降级为只读评估与告警，不得执行写操作。

#### 场景: 证据不足时阻断

- **当** 运行时证据链存在缺失或不可验证信息
- **那么** 系统必须进入只读阻断并记录可检索原因

### 需求:L2 三位一体证据门槛

系统执行 L2 镜像回滚前，必须同时满足回滚可达性、告警稳定窗口与依赖完整性三项校验。

#### 场景: 三项证据全部满足

- **当** 回滚目标可达、观察期告警稳定且依赖配置完整
- **那么** 系统必须允许进入 L2 执行

#### 场景: 任一证据不满足

- **当** 三项证据中的任意一项校验失败
- **那么** 系统必须禁止 L2 并降级至 L3 人工介入

### 需求:分级观察期矩阵

系统必须按告警类别与严重度使用分级 Soak Time，并在观察期内维持 `PendingVerify` 语义。

#### 场景: 观察期内自动恢复

- **当** 告警在观察期结束前消失
- **那么** 系统必须标记 `Suppressed` 且不执行任何写操作

### 需求:影子执行说明

系统在阻断自动动作时，必须输出“本应执行动作 + 阻断原因 + 证据摘要”的影子执行说明。

#### 场景: 命中命名空间阻断预算

- **当** 命名空间级预算触发只读阻断
- **那么** 系统必须记录原计划动作与阻断依据并可通过关联键检索
