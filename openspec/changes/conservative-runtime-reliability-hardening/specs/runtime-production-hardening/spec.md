## ADDED Requirements

### 需求:保守模式观测关联键一致性

系统在保守模式下必须保证 audit、event、metric 三类输出使用同一关联键，支持单次请求全链路检索。

#### 场景: 阻断路径关联

- **当** 请求因证据不足或预算超限被阻断
- **那么** 系统必须输出可串联的审计、事件与指标记录

### 需求:影子执行可观测输出

系统在只读阻断时必须输出影子执行信息，至少包含拟执行动作、阻断原因、证据摘要与预算命中信息。

#### 场景: 命中只读阻断

- **当** 系统阻断自动写操作
- **那么** 系统必须输出影子执行事件并更新状态摘要

### 需求:观测缺失自保护

系统在部分观测组件不可用时必须保持主流程安全降级，并记录“观测降级”状态，禁止静默失败。

#### 场景: 事件或指标上报失败

- **当** Event 或 Metric 上报失败
- **那么** 系统必须继续执行安全判定并输出观测降级证据

## MODIFIED Requirements

### 需求:执行链路去硬编码

系统必须禁止在运行时执行链路中硬编码业务阈值与时间窗口，必须由 CRD 字段或配置项驱动；保守模式新增的观察期与预算阈值同样必须配置化。

#### 场景: 门禁参数读取

- **当** 编排链路执行安全门禁判定
- **那么** 系统必须从请求配置读取幂等窗口、限频窗口、爆炸半径与熔断阈值

#### 场景: 保守参数配置化

- **当** 系统执行观察期与预算阻断判定
- **那么** 系统必须使用声明式配置参数并禁止硬编码默认业务阈值
