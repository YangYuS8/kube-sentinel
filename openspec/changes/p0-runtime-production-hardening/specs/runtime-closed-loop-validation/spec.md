## MODIFIED Requirements

### 需求:运行时闭环验收基线

系统必须提供可重复执行的闭环验收流程，并增加“参数配置化与事实输入化”强断言。

#### 场景: 演练配置化校验

- **当** 运行主线演练脚本
- **那么** 系统必须验证门禁与幂等阈值来自请求配置且与输出状态一致

### 需求:双层熔断有效性验证

系统必须验证对象级与域级熔断均可阻断后续自动写操作，并验证阈值来源于声明式配置。

#### 场景: 配置阈值触发熔断

- **当** 连续失败达到请求中配置的熔断阈值
- **那么** 系统必须阻断后续自动写操作并输出阈值证据
