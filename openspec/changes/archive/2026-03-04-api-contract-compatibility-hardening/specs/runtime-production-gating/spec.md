## ADDED Requirements

### 需求:发布判定绑定 API 契约风险
生产门禁必须将 API 契约判定纳入发布决策；当契约风险等级为高或兼容性要求未满足时必须阻断放量。

#### 场景: 契约高风险阻断发布
- **当** 发布门禁检测到 version-bump-required 但发布计划未包含版本切换窗口
- **那么** 系统必须输出 block 并禁止继续放量

### 需求:生产门禁证据补齐状态语义
生产门禁判定证据必须包含状态语义最小集合（阶段、最近动作、失败原因、下一步建议）以支持值班诊断。

#### 场景: 证据缺失下一步建议
- **当** 门禁判定结果生成但缺少下一步建议字段
- **那么** 系统必须将该判定视为不完整并阻断发布
