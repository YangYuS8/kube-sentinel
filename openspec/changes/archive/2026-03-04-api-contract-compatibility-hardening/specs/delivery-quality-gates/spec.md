## ADDED Requirements

### 需求:API/CRD/Helm 一致性门禁

质量门禁必须在发布前校验 API 定义、CRD 生成物与 Helm schema 约束的一致性；任一不一致必须阻断交付。

#### 场景: Helm 约束未同步

- **当** API/CRD 已更新但 Helm values/schema 未同步反映约束
- **那么** 系统必须输出阻断结论并给出修复路径

### 需求:兼容性判定证据输出

质量门禁必须输出 API 兼容性证据，至少包含兼容性分类、受影响字段、迁移要求与风险等级。

#### 场景: 兼容性分类缺失

- **当** 门禁执行时无法解析本次变更的兼容性分类
- **那么** 系统必须将结果标记为 block 并提示补充兼容性声明
