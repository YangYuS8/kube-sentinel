## ADDED Requirements

### 需求:提案规模治理门禁

交付质量门禁必须校验提案规模治理约束，当检测到超阈值大变更且未拆分时，必须输出阻断结论与拆分建议。

#### 场景: 超阈值但未拆分

- **当** 门禁检测到 capability 数量大于等于 3 或增量需求条目总数大于 10，且未声明拆分计划
- **那么** 门禁必须输出 block 结论并给出拆分建议

#### 场景: 超阈值且已拆分

- **当** 门禁检测到原始需求已按治理规则拆分为多个变更
- **那么** 门禁必须输出 allow 或 degrade 结论并附带拆分证据

### 需求:例外审批审计输出

交付质量门禁在放行例外场景时必须输出审批证据字段，至少包含审批人、审批原因、时间戳与关联键。

#### 场景: 例外放行

- **当** 变更命中强制拆分条件但存在有效例外审批
- **那么** 门禁必须输出审批审计字段并允许继续流程

#### 场景: 例外字段缺失

- **当** 门禁尝试放行例外但审批字段缺失
- **那么** 门禁必须输出 block 结论并提示补齐审批字段
