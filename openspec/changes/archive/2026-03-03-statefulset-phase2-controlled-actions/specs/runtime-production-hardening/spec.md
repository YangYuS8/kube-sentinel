## ADDED Requirements

### 需求:Phase 2 灰度发布控制

系统必须支持 StatefulSet Phase 2 的灰度发布控制，至少包含命名空间白名单、对象级审批与快速回滚开关。

#### 场景: 灰度范围受控

- **当** 运维仅对部分命名空间启用 Phase 2
- **那么** 系统必须确保非白名单命名空间继续保持只读策略

### 需求:Phase 2 指标门禁

系统必须为 StatefulSet Phase 2 输出误动作率、回退率、冻结触发率等关键指标，并作为发布阻断依据。

#### 场景: 指标越线回滚

- **当** 任一 Phase 2 关键指标超过阈值
- **那么** 系统必须触发自动降级或提示立即回滚到只读模式

## MODIFIED Requirements

### 需求:执行链路去硬编码

系统必须将 Phase 2 的授权、冻结与阈值参数全部配置化，禁止硬编码策略值。

#### 场景: Phase 2 参数读取

- **当** 系统执行 StatefulSet 条件可写判定
- **那么** 必须从声明式配置读取开关、白名单、审批与冻结参数
