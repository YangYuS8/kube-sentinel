## 新增需求

### 需求:V1 自动动作必须限定为 Deployment 单一 L1 尝试
系统必须将 V1 自动写动作限定为 Deployment 的单一 L1 尝试，禁止将 Deployment L2/L3 自动升级、复杂回滚编排或其他更高风险动作视为 V1 核心承诺。

#### 场景: 请求需要复杂 Deployment 自动恢复
- **当** 某个 incident 需要超出单一 L1 的 Deployment 自动恢复能力
- **那么** 系统必须停止在 V1 核心路径内继续自动升级，并转为人工建议或非核心能力处理

## 修改需求

### 需求:首发版本范围冻结
系统必须将首发可交付版本限定为 Deployment 的安全 L1 自动处置闭环，禁止在该版本中默认启用 StatefulSet 可写路径、Deployment L2/L3 自动升级或 agent 驱动的生产写动作。

#### 场景: 请求超出首发范围
- **当** 请求需要 StatefulSet 可写动作、Deployment L2/L3 自动升级或 agent 驱动写操作
- **那么** 系统必须保持只读评估或人工介入建议，不得将其视为首发验收范围内的自动动作

### 需求:首发闭环验收基线
系统必须为首发版本提供可重复验证的端到端闭环，至少覆盖告警接入、HealingRequest 映射、最小安全门禁、写前快照、Deployment L1 动作、基础审计/事件/指标输出，以及人工接管所需状态语义。

#### 场景: 首发闭环通过
- **当** Deployment 告警到达且所有前置条件满足
- **那么** 系统必须完成从告警接入到 L1 动作执行、状态更新和证据输出的完整链路
