## ADDED Requirements

### 需求:首发版本范围冻结

系统必须将首发可交付版本限定为 Deployment 的安全 L1 自动处置闭环，禁止在该版本中默认启用 StatefulSet 可写路径或 Deployment L2/L3 自动升级。

#### 场景: 请求超出首发范围

- **当** 请求需要 StatefulSet 可写动作或 Deployment L2/L3 自动升级
- **那么** 系统必须保持只读评估或人工介入建议，不得将其视为首发验收范围内的自动动作

### 需求:首发闭环验收基线

系统必须为首发版本提供可重复验证的端到端闭环，至少覆盖告警接入、HealingRequest 映射、安全门禁、写前快照、Deployment L1 动作和审计证据输出。

#### 场景: 首发闭环通过

- **当** Deployment 告警到达且所有前置条件满足
- **那么** 系统必须完成从告警接入到 L1 动作执行和证据输出的完整链路
