## ADDED Requirements

### 需求:回滚失败后基于快照恢复
系统在 L2 回滚失败时必须使用执行前快照触发恢复流程，并输出恢复开始、恢复完成或恢复失败的完整证据。

#### 场景: 回滚失败触发恢复
- **当** 系统执行 L2 回滚失败
- **那么** 系统必须立即执行基于快照的恢复并记录恢复结果

## MODIFIED Requirements

### 需求:回滚证据可观测输出
系统必须在每次 L2 回滚决策中输出候选筛选证据，并明确记录回滚执行动作与失败恢复结果；证据不足时必须明确说明降级原因；同时必须记录对应快照标识与恢复耗时。

#### 场景: StatefulSet 证据不足降级
- **当** StatefulSet 无法建立完整回滚证据链
- **那么** 系统必须禁止 L2 并记录降级到 L3 的依据

#### 场景: 回滚恢复证据可追溯
- **当** 任一工作负载发生 L2 回滚失败并触发恢复
- **那么** 系统必须输出可关联到快照标识的恢复证据链
