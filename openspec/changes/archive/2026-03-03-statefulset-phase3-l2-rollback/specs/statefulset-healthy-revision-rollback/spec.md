## ADDED Requirements

### 需求:StatefulSet L2 健康版本候选筛选
系统必须在执行 StatefulSet L2 回滚前完成候选版本筛选，且候选必须同时满足稳定窗口与依赖完整性校验。

#### 场景: 候选筛选通过
- **当** StatefulSet 存在满足健康条件的历史版本
- **那么** 系统必须将该版本记录为 L2 回滚候选并进入回滚执行阶段

#### 场景: 候选筛选失败
- **当** StatefulSet 不存在满足健康条件的历史版本
- **那么** 系统必须禁止 L2 并降级到 L3 人工介入

### 需求:StatefulSet L2 回滚执行与恢复
系统必须在 L2 执行前创建快照；回滚失败时必须恢复快照并记录失败证据。

#### 场景: L2 回滚成功
- **当** 系统执行 StatefulSet L2 回滚成功
- **那么** 系统必须更新状态为已完成并记录目标健康版本

#### 场景: L2 回滚失败
- **当** 系统执行 StatefulSet L2 回滚失败
- **那么** 系统必须恢复快照、进入冻结并回退只读

### 需求:StatefulSet L2 失败冻结与人工解锁
系统必须在 L2 失败后进入冻结窗口，冻结期间禁止任何自动写操作，直到人工解锁。

#### 场景: 冻结窗口阻断
- **当** StatefulSet 处于 L2 失败冻结窗口
- **那么** 系统必须阻断后续自动动作并输出剩余冻结时间
