## 修改需求

### 需求:预生产验证通过前禁止生产灰度

运行时闭环验证必须将预生产验证结果作为生产灰度前置条件，未通过时禁止放量并要求先完成修复验证；用于本地与测试环境的最小 smoke 回路必须可重复执行，并明确区分默认阻断路径与放宽单次测试请求后的成功路径。

#### 场景: 预生产验证失败

- **当** 闭环验证输出失败或证据不完整
- **那么** 系统必须阻断生产灰度并输出修复建议

#### 场景: 预生产验证通过

- **当** 闭环验证输出通过且证据完整
- **那么** 系统必须允许进入生产灰度放量判定

#### 场景: 本地 smoke 可重复执行

- **当** 开发者在同一测试环境中重复执行本地 smoke 验证
- **那么** 系统必须能够稳定复现默认阻断路径与成功闭环路径，并输出一致的前置条件和结果说明
