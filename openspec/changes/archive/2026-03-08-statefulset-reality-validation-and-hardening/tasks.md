# statefulset-reality-validation-and-hardening Tasks

## 1. StatefulSet 真实性验证入口

- [x] 1.1 设计并新增受显式环境变量控制的 StatefulSet 真实集群验证入口；验收标准：在未启用或前置条件不足时测试会清晰跳过，在启用时会创建最小 StatefulSet 场景并输出可检索的验证证据；单元/集成测试覆盖跳过条件与启用条件。
- [x] 1.2 为真实集群验证准备最小可复现的 StatefulSet L1/L2 场景；验收标准：测试场景能够稳定构造 revision 历史、L1 失败升级 L2 或直接验证 L2 候选筛选，不依赖人工干预；测试覆盖候选存在与候选缺失两类失败路径。

## 2. 运行时语义修正与测试补强

- [x] 2.1 根据真实性验证结果修正 StatefulSet 候选筛选、回滚或冻结判断中不成立的实现假设；验收标准：仅修改被真实集群证伪的判断条件，无法证明安全时统一降级到 L3 或冻结；单元测试覆盖真实根因对应的边界值与回退路径。
- [x] 2.2 对齐 StatefulSet L2 回滚失败后的 snapshot restore、冻结状态和人工介入证据输出；验收标准：status、event、audit 与 metrics 对成功、fallback、degraded 三类结果保持一致，真实集群验证能观察到冻结或恢复证据；测试覆盖回滚失败恢复成功、恢复失败和重复触发冻结阻断。
