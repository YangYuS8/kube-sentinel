## ADDED Requirements

### 需求:交付门禁绑定发布门禁
系统必须将质量门禁结论绑定到发布门禁判定；当质量门禁结论为 `block` 时，发布流程必须停止推进并输出阻断原因。

#### 场景: 质量门禁阻断发布推进
- **当** 统一质量门禁返回 `block`
- **那么** 系统必须拒绝进入后续发布阶段并记录阻断证据

### 需求:发布前质量证据完整性
系统必须在发布窗口结束时输出质量门禁证据摘要，至少包含检查项通过性、失败分类与恢复条件，且该摘要必须可检索。

#### 场景: 发布窗口生成质量证据摘要
- **当** 预发布或发布窗口结束
- **那么** 系统必须生成并持久化质量门禁证据摘要
