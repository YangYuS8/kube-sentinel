## 新增需求

### 需求:基础观察面必须支持 Agent 焦点判断
系统必须为 Agent v1 提供最小观察证据集合，用于支撑 `current focus` 和 `next steps` 的生成，至少覆盖动作结果、阻断原因、快照状态、最近相关事件和稳定关联键。

#### 场景: Agent 为 blocked incident 生成焦点判断
- **当** Agent 为一个 blocked incident 生成 `current focus`
- **那么** 系统必须能够提供足以区分安全阻断、启动失败、配置依赖问题或证据不足的最小观察证据

### 需求:Telegram incident card 必须可追踪到观测证据
系统必须使 Telegram incident card 能够追踪回对应的对象、事件、审计记录和趋势视图，禁止让通知成为无法继续追查的孤立消息。

#### 场景: 值班人员从 Telegram 消息继续排查
- **当** 值班人员从 Telegram 收到某个 incident 的通知卡片
- **那么** 该卡片必须包含足以定位 `HealingRequest`、Grafana 趋势视图和精确排查入口的稳定关联信息

## 修改需求

### 需求:基础观察面必须服务值班理解与人工接管
系统必须将基础观察面限定为帮助值班人员快速理解 incident、支持 Agent 生成五段式输出并完成人工接管，至少覆盖触发情况、最小动作结果、阻断原因、快照结果、最近相关事件和稳定关联键，禁止将发布治理趋势视为 V1 基础观察面的前置内容。

#### 场景: Agent 或值班人员查看单个 incident 的观察证据
- **当** Agent 或值班人员查看某个 incident 的运行证据
- **那么** 系统必须提供与该 incident 相关的基础事件、审计记录、关键指标和关联键
