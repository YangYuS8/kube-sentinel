## 新增需求

### 需求:闭环演练必须验证 oncall state
系统必须让本地与测试环境的闭环演练同时验证 runtime phase 和 oncall state，禁止仅凭内部 phase 断言就宣称值班体验正确。

#### 场景: 本地 smoke 验证值班语义
- **当** 开发者在本地或测试环境执行夜间值班闭环演练
- **那么** 演练结果必须同时说明真实 phase 和对外 oncall state

## 修改需求

### 需求:本地 smoke 可重复执行
系统必须能够稳定复现默认阻断路径、观察路径与成功闭环路径，并输出一致的前置条件、真实 phase 和 oncall state 说明，而不只是原始状态机结果。

#### 场景: 本地重复执行 smoke
- **当** 开发者在同一测试环境中重复执行本地 smoke 验证
- **那么** 系统必须能够稳定输出 phase 与 oncall state 的对齐结果
