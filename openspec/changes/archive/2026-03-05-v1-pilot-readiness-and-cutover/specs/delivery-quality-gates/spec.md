## ADDED Requirements

### 需求:pilot 批次放量前置检查

交付质量门禁必须在每个 pilot 批次放量前校验质量结果、证据完整性与阶段合法性，任一不满足必须阻断。

#### 场景: 放量前置检查失败

- **当** 质量检查失败或证据字段不完整
- **那么** 门禁必须输出 `block` 并拒绝进入该批次放量

#### 场景: 放量前置检查通过

- **当** 质量检查通过且证据字段完整
- **那么** 门禁必须输出 `allow` 并标记可进入下一批次

### 需求:cutover 决策字段稳定输出

交付质量门禁必须输出与 cutover decision pack 对齐的稳定字段命名与语义，禁止同名异义。

#### 场景: 连续执行字段一致

- **当** 连续多次执行同一批次门禁
- **那么** 输出字段名称和语义必须保持一致

#### 场景: 字段语义冲突

- **当** 门禁字段与决策包语义不一致
- **那么** 系统必须阻断并提示 `decision_contract_mismatch`
