## ADDED Requirements

### 需求:运行时链路关联键
系统必须在自愈请求全链路中维护统一关联键，用于串联 status、event、metric 与审计记录。

#### 场景: 自愈动作执行成功
- **当** 某 Deployment 完成一次自动处置
- **那么** 系统必须在所有观测输出中包含同一关联键

## MODIFIED Requirements

### 需求:v1alpha1 Deployment 自愈编排范围
系统必须在 v1alpha1 阶段仅对 Deployment 执行自动自愈动作，禁止对 StatefulSet 等其他工作负载直接执行写操作；并且在运行时事件接入场景下，必须在入口阶段完成类型判定。

#### 场景: 非 Deployment 目标被触发
- **当** 告警事件指向的目标资源类型不是 Deployment
- **那么** 系统必须拒绝执行自愈写操作，并记录“类型不在 v1alpha1 支持范围”的审计事件

#### 场景: 运行时入口即判定不支持类型
- **当** Webhook 映射阶段识别到非 Deployment 类型
- **那么** 系统必须终止写操作链路并输出只读评估结果

### 需求:编排链路幂等执行
系统必须保证同一告警在重复触发时不会导致重复副作用，必须以目标对象当前状态作为判定依据，并在重复事件场景保持阶段推进稳定。

#### 场景: 同一告警重复到达
- **当** 在幂等窗口内接收到同一 Deployment 的重复告警
- **那么** 系统必须复用当前策略阶段与执行记录，禁止重复创建冲突动作

#### 场景: 重复事件在阶段切换点到达
- **当** 第二次事件到达时第一条链路已推进到更高阶段
- **那么** 系统必须维持已推进阶段，禁止回退到低阶段重复执行
