## ADDED Requirements

### 需求:熔断触发可视化
系统必须在熔断触发时输出熔断层级、触发阈值、当前计数与预计恢复时间。

#### 场景: 熔断被触发
- **当** 对象级或域级熔断进入开启状态
- **那么** 系统必须记录触发依据并输出预计恢复时间

## MODIFIED Requirements

### 需求:域级熔断
系统必须实现域级熔断（命名空间或全局）：当失败密度与影响面超过阈值时，必须暂停该范围的新自愈写操作；并在运行时告警洪峰下优先保护系统稳定性。

#### 场景: 命名空间出现集中失败
- **当** 某命名空间内在统计窗口期内出现超过阈值的熔断事件或失败密度
- **那么** 系统必须触发域级熔断并拒绝该范围内新的自动写操作请求

#### 场景: 告警洪峰下的域级保护
- **当** 短时间内多个对象同时触发失败并逼近域级阈值
- **那么** 系统必须优先进入域级熔断以阻断后续自动写操作扩散

### 需求:分级治理优先级
系统必须遵循双层熔断优先级：域级熔断生效时，对象级状态不得绕过域级限制；并且对象级恢复不应自动解除域级约束。

#### 场景: 对象级恢复但域级仍熔断
- **当** 某 Deployment 的对象级熔断已恢复但所在域级熔断仍未解除
- **那么** 系统必须继续拒绝该对象自动写操作，直到域级熔断解除

#### 场景: 多对象混合恢复
- **当** 域内部分对象已恢复但整体域级条件未满足
- **那么** 系统必须维持域级熔断并拒绝域内写操作
