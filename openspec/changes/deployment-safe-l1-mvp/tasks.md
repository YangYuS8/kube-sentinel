## 2. Deployment L1 安全闭环

- [ ] 2.1 在编排链路中固定 Deployment 首发 L1 单动作成功路径：门禁通过后创建写前快照，再执行唯一 L1 动作并写入成功证据；验收标准：满足前置条件时状态进入成功阶段并记录最近动作、门禁裁决和审计结果；测试：补充 orchestrator 单元测试覆盖成功路径与幂等窗口内重复触发阻断。
- [ ] 2.2 固定 Deployment 首发 L1 失败路径：门禁失败、快照失败或 L1 动作失败时禁止自动升级，输出阻断原因和下一步建议；验收标准：失败场景不会继续执行更激进动作，状态字段可解释且审计证据完整；测试：补充 orchestrator 与 snapshot 相关单元测试覆盖维护窗口、速率限制、爆炸半径、熔断、快照失败和 L1 执行失败。
