## ADDED Requirements

### 需求:L2 回滚结果门禁权重
系统必须将 L2 回滚结果纳入发布门禁权重计算，并在连续窗口内对失败回退与降级比例执行阻断判定。

#### 场景: L2 回滚失败率越线阻断
- **当** 连续窗口内 L2 回滚失败回退比例超过阈值
- **那么** 系统必须触发发布阻断并输出降级建议

## MODIFIED Requirements

### 需求:Deployment L2 候选筛选与恢复证据
系统必须在 Deployment L2 回滚前完成候选筛选与依赖校验，并在回滚失败时输出快照恢复证据链；同时必须输出该次 L2 结果对发布门禁判定的影响。

#### 场景: Deployment L2 回滚失败触发恢复
- **当** Deployment L2 回滚执行失败
- **那么** 系统必须基于快照执行恢复并记录恢复结果

#### 场景: Deployment L2 结果影响门禁判定
- **当** Deployment L2 完成一次回滚执行
- **那么** 系统必须输出该次结果对发布门禁的影响说明
